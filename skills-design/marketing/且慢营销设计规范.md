# 且慢营销设计规范

> 本规范用于指导且慢品牌的营销类视觉设计（H5/落地页/运营位/海报/信息流/战报等），在延续且慢 UI 设计系统的基础上，补齐营销场景的**信息层级、模块模板、出图规格、合规与质检**要求；并提供**H5 模块模板库（积木化复用）**与推荐拼装方案，做活动页时按「主题 → 结构 → 组件」拼装即可。

---

## 1. 目标与适用范围

### 1.1 目标

- **一致**：营销物料与产品内 UI 视觉同源，用户一眼识别“且慢”。
- **清晰**：优先保证信息可读与层级明确，降低理解成本。
- **可信**：金融属性下的理性克制，避免“强促销”视觉误导。
- **可复用**：形成固定组件与版式套路，提升批量产出效率。

### 1.2 适用范围

- **页面**：活动 H5、落地页、长图页、专题页、战报页
- **运营位**：Banner、弹窗、卡片、引导浮层、站内信配图
- **投放**：信息流素材、短视频封面、开屏/贴片静帧（按渠道规范适配）

---

## 2. 且慢品牌营销气质（基调）

### 2.1 关键词

- **专业可信**、**克制清爽**、**数据友好**、**温和不压迫**、**理性但不冷**

### 2.2 视觉优先级（强制）

1. **信息清晰**（一眼读懂是什么/对谁/有什么价值）
2. **层级明确**（标题 > 关键数据/利益点 > 解释 > 免责声明）
3. **一致性**（颜色、字号、圆角、卡片、按钮规则统一）
4. **装饰点缀**（仅用于增强识别与节奏，不能影响阅读）

### 2.3 禁止/避免

- **高饱和撞色**、霓虹渐变、厚重投影、过强纹理背景
- 夸张“促销风”组件：爆炸贴、粗描边大字、强烈红黄对比堆叠
- 情绪化拟人插画（过强表情/戏谑风）与与金融信任感冲突的视觉语言

## 3. 视觉系统（与 UI 规范同源）

> 本章节的色彩/排版/间距来自 `qieman skills/SKILL.md`（且慢 UI 设计规范）。营销侧必须复用，避免自创体系导致“像别的品牌”。

### 3.1 色彩系统

#### 3.1.1 主色

- **Brand Primary**：`#1B88EE`（按钮、链接、关键高亮）
- **页头海报背景**：`#3180EC`（首屏/头图大面积背景）

#### 3.1.2 文本颜色层级（必须遵循）

- **重要标题/正文**：`#333333`
- **正文/说明**：`#606060`
- **辅助说明/注释**：`#999999`
- **弱提示/禁用**：`#CCCCCC`
- **深色背景反白**：`#FFFFFF`

#### 3.1.3 功能色（少量、精准）

- **成功/正向**：`#07AD8F`
- **提醒**：`#EA9500`
- **警示/风险**：`#FA440C`

#### 3.1.4 背景/卡片/边框（营销常用容器）

- **页面底色**：`#F9FAFB`
- **卡片默认**：`#FFFFFF`
- **信息淡底**：主 `#F0F6FF` / 提醒 `#FFFAEF` / 警示 `#FEEDE9`
- **默认边框**：`#D8D8D8`；提示边框：蓝 `#9CCBF8` / 黄 `#FFDC9E` / 红 `#FAB6A5`

### 3.2 排版系统（营销可直接用的取值）

- **主标题**：32pt（首屏/头图）
- **分区标题**：26pt / 24pt / 19pt（按层级递减）
- **正文**：16pt（默认）
- **说明**：14pt / 12pt（备注、口径、免责声明）

#### 3.2.1 营销头图主标题与副标题样式（参考图 1 / 图 2）

头部氛围图（含图标类、人物类）左侧文字区须遵循以下**文字大小与排版模式**，保证与参考图一致、层级清晰。

| 层级 | 字号 | 字重 | 颜色 | 行距/间距 |
|------|------|------|------|-----------|
| **主标题** | **26pt**（或 28pt） | 加粗（600–700） | `#333333` | 与副标题间距 **8px**（或 0.3em） |
| **副标题** | **16pt**（或 18pt），约主标题 **60%～70%** | 常规（400） | `#606060` | 与备注间距 **4px**（或 0.2em） |
| **备注文案** | **14pt**（或 12pt），约副标题 **70%～80%** | 常规（400） | `#999999` | — |

**排版约定**：

- **对齐**：主标题、副标题、备注均**左对齐**，与 Logo/品牌区同侧；首屏头图区域左侧文字与右侧氛围图**顶对齐**（布局使用 `align-items: flex-start`），保持视觉整齐。
- **垂直间距**：主标题与副标题之间用**较大间距**（8px 或 0.3em），副标题与备注之间用**较小间距**（4px 或 0.2em），层级分明。
- **色彩**：主标题深色（`#333333`）保证对比度，副标题/备注逐级减弱（`#606060`、`#999999`），适配浅色背景。
- **字数**：主标题 1 行、副标题 1–2 行、备注 0–1 行，避免堆叠。

**CSS 示例**：

```css
.bill-header .text-area h1 { font-size: 26px; font-weight: 600; color: #333333; margin-bottom: 8px; }
.bill-header .text-area .sub { font-size: 16px; font-weight: 400; color: #606060; margin-bottom: 4px; }
.bill-header .text-area .remark { font-size: 14px; font-weight: 400; color: #999999; }
```

### 3.3 间距与圆角（统一手感）

- **模块间距**：16px
- **卡片内边距**：12px
- **圆角**：8px（小）/ 12px（中）/ 24px（大，少量）

#### 3.4.4 使用原则

- **一屏/一模块内**：同一背景区域只使用同一套 Logo（全蓝或全白），不混用。
- **清晰度**：优先使用 SVG；PNG 建议 2x 以上分辨率，避免虚边。
- **口号**：「安放财富 · 静待花开」与且慢 Logo 组合使用时，保持字色与 Logo 主色一致（浅底用蓝，深底用白）。

## 4. 营销信息结构（内容与视觉的配合）

### 4.1 标准信息层级（建议顺序）

1. **标题**：一句话说清“是什么/对谁/有什么价值”
2. **关键利益点**：最多 3 条（短句、可编号/轻图标）
3. **关键数据/证明**：一屏只讲一个结论（必要时给口径）
4. **行动引导（CTA）**：明确且稳定（主按钮为主）
5. **风险提示/限制**：可见但不抢夺（小字号 + 次级颜色）

### 4.2 数字与口径

- **单位统一**：金额、百分比、期限等格式统一（同页不混用“元/万元/¥”）
- **时间明确**：数据口径必须有时间范围（如“截至 2026-01-29”）
- **对比清晰**：同比/环比/对比对象写清楚，避免误读

## 5. 营销组件与模块模板（可直接套用）

### 5.1 首屏/页头海报（H5/落地页通用）

- **布局**：左右排版（建议关键视觉区域保持 4:3）
- **背景**：`#3180EC`；浅色头图时见《营销头部氛围图设计 SKILL》
- **文案**：白字 `#FFFFFF`（深色底）；浅色底时见 3.2.1
- **结构**：
  - 左：主标题 + 副标题 + 关键信息（可 1 行）
  - 右：简化插图/图表/装饰几何（避免复杂写实）
- **主副标题样式**：采用**营销头图**布局（左文右图、浅色背景）时，须遵循 **3.2.1 营销头图主标题与副标题样式**（参考图 1 / 图 2）：主标题 26pt 加粗、副标题 16pt 常规、左对齐、层级间距 8px / 4px。
- **注意**：首屏信息只允许一个主结论，避免堆满卖点。

### 5.2 摘要卡（“一眼看懂”）

- **底色**：`#DBEBFF`
- **边框**：`1px solid #9CCBF8`
- **圆角**：12px
- **内容结构**：结论（大字）> 关键数据（对齐）> 解释（小字）

### 5.3 内容卡（承载卖点/规则/FAQ）

- **底色**：`#FFFFFF`（必要时加 `1px #D8D8D8`）
- **圆角**：8px
- **投影**：弱投影或无投影优先（保持克制）

### 5.4 按钮与 CTA（转化关键）

- **主按钮**：背景 `#1B88EE` / 文字 `#FFFFFF` / **胶囊形**（`border-radius: 9999px`）/ 内边距 `12px 24px` / **无边框、无投影**
- **次按钮/文字按钮**：文字 `#1B88EE`，可用轻底 `rgba(27,136,238,0.1)` 作为 hover/弱强调
- **规则**：
  - 一屏最多一个主 CTA
  - CTA 文案优先动词 + 结果（例如“立即测算”“查看方案”）

### 5.5 风险/提示卡（合规信息容器）

- **提醒**：底 `#FFFAEF` + 边 `#FFDC9E`
- **警示**：底 `#FEEDE9` + 边 `#FAB6A5`
- **文本**：正文用 `#606060`，关键限制可用 `#333333` 加粗，但不要红字堆叠。

### 5.6 图表/战报模块（用数据说话）

- **配色**：使用 UI 规范 `chart01 ~ chart12`，同页系列保持顺序一致
- **原则**：
  - 一个图只表达一个结论（标题直接说结论）
  - 高亮只高亮“结论对象”（不要全图都彩色）
  - 标注口径（时间/单位/来源）放在图下方 12–14pt

## 6. 插画/图标/摄影（风格边界）

### 6.1 图标与图标库

营销 H5、落地页、运营位等**统一使用 [Remix Icon](https://remixicon.cn) 图标库**，保证风格一致、维护简单。

#### 6.1.1 引入方式（CDN）

在页面 `<head>` 中引入 Remix Icon 字体样式：

```html
<link href="https://cdn.jsdelivr.net/npm/remixicon@4.2.0/fonts/remixicon.css" rel="stylesheet">
```

或使用国内镜像（按需选用）：

```html
<link href="https://unpkg.com/remixicon@4.2.0/fonts/remixicon.css" rel="stylesheet">
```

> 版本号可按 [Remix Icon 官网](https://remixicon.cn) 或 npm 最新版本更新；建议锁定主版本号以保证上线一致性。

#### 6.1.2 使用方式

在 HTML 中用 `<i>` 标签 + 类名 `ri-图标名` 使用图标，例如：

```html
<i class="ri-chart-line"></i>
<i class="ri-wallet-3-line"></i>
<i class="ri-arrow-right-s-line"></i>
```

图标名称与分类可在官网搜索：<https://remixicon.cn>。

#### 6.1.3 且慢营销约定（必须遵循）

- **风格**：线性风格优先（优先选用 `-line` 系列），尺寸建议 **16 / 20 / 24 / 32px** 级别。
- **颜色**：仅用三档，与规范一致——  
  `#333333`（重要/标题旁）、`#999999`（辅助/说明）、`#1B88EE`（链接/强调）。**避免彩虹图标**。
- **数量**：同一模块内图标风格、线宽统一；同页图标种类不宜过多，保持克制。

### 6.2 插画

- 简化几何、抽象数据元素、轻量金融隐喻（曲线、柱状、网格）优先
- 避免强情绪化、戏谑夸张的人物表情与姿态

### 6.3 背景与装饰

- 允许：低对比网格/点阵/几何块（强度要“看得见但不抢字”）
- 禁止：高对比纹理、强噪点、影响文字边缘的复杂背景

---

## 7. 规格与交付（面向生产）

### 7.1 命名规范（建议）

`渠道_活动名_模块_尺寸_版本_日期`

示例：`AppBanner_春节理财_首屏_1080x420_v03_20260129`

### 7.2 输出格式

- **投放/传播**：PNG（优先）/ JPG（压缩场景）/ WebP（渠道允许时）
- **可编辑源文件**：Figma/Sketch/PSD/AI（按团队协作约定）
- **页面交付**：HTML + CSS + 资源（图片按模块分目录）

### 7.3 适配与安全区

- 移动端关键文案与 CTA 必须在安全区内（考虑刘海/底部手势条）
- 多机型验证：标题不截断、按钮不贴边、行高不拥挤

---

## 8. 发布前质检清单（必过）

- **一致性**：颜色/字号/圆角是否严格使用且慢体系（尤其主蓝与灰阶）
- **字体**：营销文字是否采用阿里普惠体（3.2.0），或已声明备选方案
- **层级**：标题/利益点/数据/说明/免责声明是否一眼分清
- **可读性**：对比度足够，小字不低于 12pt，浅底深字/深底白字
- **转化链路**：CTA 明确、位置稳定、一屏不多主按钮
- **合规**：风险提示/限制/免责声明是否可见且口径正确
- **数据口径**：时间范围、单位、对比对象是否完整
- **克制**：是否存在“为了热闹而装饰”的元素（能删就删）

## 9. H5 模块模板库（积木化复用）

> 用法：每次做活动页，从下列模块里按「**主题 → 结构 → 组件**」拼装。  
> **推荐默认结构**：Hero → 卖点卡×2~3 → 流程卡 → 证明/对比卡 → FAQ/规则 → CTA → 风险提示。

### 9.1 模块 01｜Hero 首屏模块（标题 + 核心利益点）

**目的**：3 秒内让用户看懂「我能得到什么」。

| 类型 | 说明 |
|------|------|
| **组件结构** | 顶部主题图/氛围（背景层）→ 标题（1 行）+ 副标题（1–2 行）→ 标签/徽章（可选：限时/专享/新手/会员）→ 主 CTA（可选：强转化页建议放首屏） |
| **文案坑位** | 主标题：`一句话利益点（动词+结果）`；副标题：`适用人群 + 条件/时间`；CTA：`立即领取 / 去开通 / 开始测算` |
| **推荐尺寸** | 内容容器宽 343px，内边距 16px，模块间距 16/24px |

### 9.2 模块 02｜卖点卡（Benefit 图文卡）

**目的**：把「好处」拆成 1 张卡讲清。

| 类型 | 说明 |
|------|------|
| **组件结构** | 顶部插画/图标（可选）→ 标题（8–14 字）→ 要点 2–3 条（每条 12–16 字）→ 辅助说明（小字，可选） |
| **适用** | 权益、功能上新、活动利益点 |
| **注意** | 避免纯口号，尽量用「机制/证据/条件」 |

### 9.3 模块 03｜数字证据卡（Data Proof）

**目的**：增强可信度（数据/口碑/规模/覆盖）。

| 类型 | 说明 |
|------|------|
| **组件结构（推荐三列）** | 数字（大）+ 单位（小）→ 解释（1 行）→ 可选：来源/限定说明（小字） |
| **示例坑位** | `xx 万+` `用户选择`；`xx%` `覆盖/提升`；`xx 项` `服务/能力` |
| **合规提示** | 数字必须可解释，必要时加「截至 xx」 |

### 9.4 模块 04｜流程/玩法模块（Step How-to）

**目的**：降低理解成本，告诉用户怎么做。

| 类型 | 说明 |
|------|------|
| **组件结构** | 模块标题（1 行）→ 步骤 3–5 个（编号/时间轴/卡片）→ 每步：标题（6–10 字）+ 解释（1 行） |
| **适用** | 领取/开通/参与活动/完成任务 |
| **推荐** | 3 步最有效，最多 5 步 |

### 9.5 模块 05｜权益清单模块（List）

**目的**：快速扫读权益点（适合长权益/多功能）。

| 类型 | 说明 |
|------|------|
| **组件结构** | 模块标题 → 列表项（icon + 标题 + 描述）→ 可选：右侧 tag（如「限时/会员/新手」） |
| **推荐** | 每屏 4–6 项；过多则分页/折叠 |

### 9.6 模块 06｜对比模块（Why us / A-B）

**目的**：用对比强化选择理由。

| 类型 | 说明 |
|------|------|
| **组件结构** | 2 列（且慢 vs 其他）或 3 列（不同权益档）→ 行：能力/体验/服务点（对勾/叉/标签） |
| **适用** | 会员权益、服务升级、策略方案对比 |
| **注意** | 避免攻击式文案；描述要客观中性 |

### 9.7 模块 07｜口碑/案例模块（Social Proof）

**目的**：增强信任（评价、案例、媒体背书）。

| 类型 | 说明 |
|------|------|
| **组件结构** | 卡片轮播或列表 → 每条：头像/昵称（可匿名）+ 评价一句话 → 可选：评分/来源 |
| **合规提示** | 避免虚构具体收益；强调体验/服务即可 |

### 9.8 模块 08｜FAQ 折叠模块（Accordion）

**目的**：承接疑虑，减少跳失。

| 类型 | 说明 |
|------|------|
| **组件结构** | 模块标题 → 折叠项 4–8 条（默认展开 1 条） |
| **常用问题** | 适用对象/门槛、有效期、领取次数、到账方式、如何使用、失败原因 |
| **动效** | 200–300ms 高度过渡 |

### 9.9 模块 09｜规则说明模块（Activity Rules）

**目的**：把关键规则「结构化」，方便查阅。

| 类型 | 说明 |
|------|------|
| **组件结构（推荐信息块）** | 时间：`YYYY.MM.DD - YYYY.MM.DD`；对象：`新用户/会员/指定渠道`；门槛：`开通/绑卡/完成任务`；次数：`每日/每人/总量`；发放：`即时/次日/xx工作日`；失效：`过期/撤销/退款影响` |
| **建议** | 用表格式信息块，不要大段文字 |

### 9.10 模块 10｜主 CTA 模块（转化区）

**目的**：让用户明确下一步。

| 类型 | 说明 |
|------|------|
| **组件结构** | 主按钮（唯一主动作）→ 辅助说明（小字 1–2 行，如「预计 xx 秒完成」）→ 可选：次按钮（仅在必要时） |
| **按钮状态** | 默认：primary；按压：pressed；不可用：灰阶 + 限制说明 |

主按钮须遵循本章第 5.4 节：背景 `#1B88EE`，文字 `#FFFFFF`，胶囊形（`border-radius: 9999px`），内边距 12px 24px，无边框、无投影。

### 9.11 模块 11｜风险提示模块（合规必备）

**目的**：满足披露要求，避免信息压垮转化。

| 类型 | 说明 |
|------|------|
| **组件结构** | 标题：`风险提示` → 正文：默认展示 3–6 行 → 展开：`展开查看全文` |
| **推荐** | 放在 CTA 下方；若合规强要求可放 CTA 上方但折叠 |

### 9.12 模块拼装推荐方案（直接可用）

**A. 强转化活动页（最常用）**

```
Hero（带 CTA）→ 卖点卡×2 → 流程卡 → 规则（折叠）→ CTA（再次出现）→ 风险提示
```

**B. 功能宣发页（信息更完整）**

```
Hero → 卖点卡×3 → 权益清单 → 对比/证明 → FAQ → CTA → 风险提示
```

**C. 高净值/会员尊享页（更克制更高级）**

```
深色 Hero → 数字证据 → 服务权益清单 → 案例/口碑 → FAQ → CTA → 风险提示
```

### 9.13 模块可视规范（统一节奏）

| 项目 | 取值 |
|------|------|
| 内容宽 | 343px |
| 左右边距 | 16px |
| 模块间距 | 16px（常规）/ 24px（大模块分段） |
| 卡片内边距 | 16px |
| 圆角 | 12px（常规）/ 16px（高净值） |
| 阴影 | 轻（不建议重投影） |
| 主色使用 | 只用在「CTA / 关键数字 / 标题点睛」 |

与本章第 3、5 节不一致时，以第 3、5 节视觉规范为准；本节侧重**结构与模块拼装**。
