# 且慢头部氛围图设计规范（Header Atmosphere Image）

> 本规范基于「且慢 AI 3D 素材」资产库，用于指导 H5、落地页、营销专题页的**首屏/头部氛围图**设计与拼装。与《且慢营销设计规范》《且慢营销配色规范》配合使用。

---

## 1. 概述与适用范围

### 1.1 目标

- **统一**：头部氛围图与且慢品牌视觉同源，用户一眼识别「且慢」。
- **模块化**：按「背景 + 人物/风景 + 图标」积木化拼装，可快速适配不同活动主题。
- **克制**：氛围服务于信息，不抢夺标题与 CTA 的注意力。

### 1.2 适用范围

- H5 首屏头图
- 落地页 Hero 区
- 营销专题页头部
- Banner / 开屏图（按渠道规范适配）

### 1.3 素材来源

- **Figma**：[且慢 AI 3D 素材](https://www.figma.com/design/nfwWCw372v1Sy1pvOd36JO/%E4%B8%94%E6%85%A2AI-3D%E7%B4%A0%E6%9D%90)
- 导出格式：PNG（透明底）/ SVG（图标类）
- 命名建议：`头部氛围图_分类_主题_尺寸_版本`

## 2. 头图布局规则（强制）

> 头部氛围图需遵循以下布局约定，保证**文本优先、层级清晰**。

### 2.1 分区约定

- **文字与氛围图顶对齐（强制）**：左侧文字块（含 Logo/主标题/副标题）与右侧氛围图（图标或人物）在垂直方向上**顶对齐**，保持视觉整齐；实现时头部容器使用 `align-items: flex-start`，避免文字与图片中线或底对齐导致的错位感。

| 区域 | 用途 | 说明 |
|------|------|------|
| **左侧** | 预留主题文字 | 主标题、副标题、关键信息在此区域展示，为**主视觉区**；主标题**可自适应换行**，超出时折行展示，避免与右侧氛围图重叠；实现时左侧文字区使用 `flex: 1; min-width: 0`，主标题不设 `white-space: nowrap`，可选 `overflow-wrap: break-word`；窄屏时用 `font-size: clamp(18px, 5.2vw, 26px)` 自适应字号 |
| **右上角** | 营销图（氛围图） | 3D 图标、人物或装饰元素，作为**次要视觉**，不抢夺文本 |

### 2.2 背景与文本

- **背景**：**浅色系**为主（如浅蓝、浅紫、浅灰白渐变），避免深色压字
- **突出文本**：背景干净、低对比，确保标题与副标题**易读、醒目**
- **留白**：左侧及中部保留充足留白，不堆叠装饰元素

### 2.3 留白区域约定

- 左侧留白 ≥ 40% 画布宽度，专供主题文字
- 营销图/氛围图仅占右上角或右侧局部，不侵入文字区
- 整体构图以**文字为主、氛围为辅**，信息层级清晰

### 2.4 示意图（典型布局）

```
┌─────────────────────────────────┐
│  主标题                          │  ← 左侧：主题文字区
│  副标题              ┌──────┐   │
│  关键信息            │ 营销 │   │  ← 右上角：氛围图
│  （留白充足）        │ 氛围 │   │
│                     │ 图   │   │
└─────────────────────────────────┘
     浅色系背景，突出文本
```

### 2.5 参考图说明（位置 / 比例 / 高度）

调用图标与人物时，**必须分别参考图 1、图 2**，不得混用同一套比例。

| 参考图 | 适用类型 | 位置、比例与高度约定 |
|--------|----------|----------------------|
| **图 1** | 图标类头图 | **位置**：画面**右上象限**，整体位于右侧上半区；水平方向靠近右边缘（左缘约 58%、右缘留约 10%）；垂直方向顶部距画布顶部留有充足留白，底部在**画面水平中线之上**。**比例**：图标组约占画布宽度的 **20%～26%**、高度的 **20%～26%**（约 1/5～1/4 画布），在右上角区域克制呈现、不喧宾夺主，留白充足。**高度**：图标顶部与主标题齐平或略高，底部在水平中线之上，在右上半区垂直居中，下方留白充足 |
| **图 2** | 人物类头图 | **位置**：画面**右侧中右区**；人物左肩约距画布左缘 65%～70%，右肩约 85%～90%；头顶约距画布顶部 **20%**。**比例**：人物（头至腰）高度约占画布总高度的 **50%～55%**；肩宽约 **25%～30%** 画布，整体人物元素（含手持物）约占 **30%～45%** 画布宽度。**高度**：主标题基线与人物**眼睛/额头**同高，人物头部与主标题齐平，躯干向下延伸至画面中段，整体占比较大、辨识度高 |

图标偏紧凑、偏上；人物偏饱满、偏下，二者在视觉锚点与占比上明显区分。

### 3.1 背景（营销氛围图）

| 项目 | 说明 |
|------|------|
| **用途** | 头部底图，承载人物、图标等前景元素 |

| **配色** | 文本为主的头图宜用**浅色系**（浅蓝、浅紫、浅灰白渐变）；深色底需保证文字对比度 |

- 主视觉清爽：`#1B88EE → #67B0F4`
- 科技紫：`#7477D8 → #BB99FF`
- 稳重深蓝：`#061D32 → #125EA5`

### 3.2 扁平人物（Flat Characters）

| 项目 | 说明 |
|------|------|
| **风格** | 扁平化 + 轻微 3D 感，现代、专业、亲和 |
| **用途** | 强化场景感，传递「理财/分析/陪伴」等主题 |
| **数量** | 按需选用 0–1 个人物，避免多人堆叠 |

**位置、比例与高度（参考图 2）**：

| 维度 | 约定 | 说明 |
|------|------|------|
| **位置** | 画面**右侧中右区** | 人物左肩约 65%～70%、右肩约 85%～90% 处；头顶距画布顶部约 20%；主标题基线与人物眼睛/额头同高 |
| **比例** | 高约 **50%～55% 画布**（头至腰）、宽约 **30%～45% 画布**（含手持物） | 人物为主体，肩宽约 25%～30%，整体占比较大，保证辨识度与场景感 |
| **高度** | 头部与主标题齐平，躯干向下延伸至画面中段 | 人物自头部向下延伸，与图标类（图标偏上、底在中线之上）在视觉锚点上明显区分 |

调用人物类头图时，**位置 / 比例 / 高度**以**图 2**为准，避免与图标类头图混用同一尺寸。

**典型组合**（供参考）：

| 类型 | 场景 | 元素示意 |
|------|------|----------|
| 分析型 | 投研、报告、策略 | 深色西装、放大镜、浅蓝渐变背景 |
| 工作型 | 定投、交易、陪伴 | 坐姿、笔记本、仙人掌、图表线条、浅黄绿渐变 |
| 移动型 | 触达、通知、服务 | 蓝色西装、手机、通知图标、紫蓝背景 |

**注意**：避免强情绪化、戏谑夸张的表情与姿态；与金融信任感保持一致。

**背景色按人物衣服颜色定义（强制）**：

头图背景须根据人物**主色（衣服颜色）**选取对应渐变，使画面风格融合。在 `assets-config.js` 中为每张人物图指定 `bgTheme`，或在 H5 中按人物主色映射：

| 人物主色（衣服/配饰） | bgTheme | 背景渐变（配色规范） |
|----------------------|---------|----------------------|
| 浅蓝、天蓝、蓝 polo | `people` | 品牌蓝 A100–A200：`#F5F9FF → #DBEBFF → #F5F9FF` |
| 紫色、紫蓝西装/上衣 | `people-purple` | 科技紫 100–200：`#F5F0FF → #E7DBFF → #F5F0FF` |
| 深蓝、藏青、深色西装 | `people-deep` | 稳重深蓝 B100–B200：`#F0F6FF → #DBEBFF → #F0F6FF` |
| 白/灰/中性色 | `people` | 品牌蓝 A100–A200（默认融合） |
| 暖色、米黄、浅棕 | `people-warm` | 暖金 100：`#FFF7E5 → #FCFDFD → #FFF7E5` |

> 实现方式：在 `assets-config.js` 的 PEOPLE 中，可将每张图与 `bgTheme` 成对配置；或扩展 `getRandomWithBg` 支持按文件名返回对应 `bgTheme`，以使人像背景色随衣服颜色动态适配。

**背景映射执行流程（资产 + 前端共同维护，强制）**：

1. **逐张记录上衣主色**：在 `assets/人物/` 目录中建立表格（推荐保存在 README.md 内），逐张填写「文件名、人物上衣主色、建议 `bgTheme`」。示例：

| 文件名 | 上衣主色 | 必填 `bgTheme` |
|--------|----------|----------------|
| `Frame 1000009047.png` | 浅蓝衬衫 | `people` |
| `Frame 1000009048.png` | 深蓝外套 | `people-deep` |
| `Frame 1000009049.png` | 紫色西装 | `people-purple` |
| `Frame 1000009050.png` | 暖米色毛衣 | `people-warm` |
| `Frame 1000009051.png` | 天蓝 Polo | `people` |
| `Frame 1000009052.png` | 藏青西装 | `people-deep` |
| `Frame 1000009053.png` | 紫蓝衬衫 | `people-purple` |
| `Frame 1000009054.png` | 浅棕针织 | `people-warm` |
| `Frame 1000009055.png` | 浅蓝夹克 | `people` |

2. **同步至 `assets-config.js`**：在 `PEOPLE_BG` 映射中逐项填写，严禁全部指向 `people`。如人物上衣为暖色，必须写成 `'人物/Frame 1000009050.png': 'people-warm'`。
3. **开发联调**：调用 `ASSETS.getRandomWithBg('people')` 后，头图 `header` 的 `data-bg` 必须出现与衣服颜色一致的 `people-*`，否则视为不合格。
4. **质检**：在浏览器中刷新 3 次以上，确认不同人物会触发对应背景渐变（浅蓝→蓝、紫衣→紫、暖衣→金），截图归档以备查。

**资产路径**（按 `assets-config.js` 约定）：人物图存放于 `assets/人物/`，当前纳入：`Frame 1000009047.png` 至 `Frame 1000009055.png`（9 张），可在该文件夹新增或替换 PNG 后于配置中更新路径；新增时须注明人物主色并配置对应 `bgTheme`。

## 4. 拼装与构图

### 4.1 拼装顺序（自底向上）

1. **背景层**：铺底 375×H 渐变或纯色
2. **风景/装饰层**（可选）：若使用风景，置于背景之上、人物之下
3. **人物层**（可选）：单人物，居左或居右，留出文字区
4. **图标层**（可选）：1–3 个图标点缀，不遮挡标题与 CTA

### 4.2 构图原则

- **留白**：左侧保留至少 40% 空间给主题文字（与 2.3 留白区域约定一致）
- **层级**：标题 > 副标题 > 氛围图；氛围图不压过文字
- **对齐**：与《且慢营销设计规范》5.1 首屏结构一致：左文右图，或左图右文，关键区域约 4:3

### 4.3 与首屏文案的配合

- 背景与人物/图标的色调需与首屏主色（`#3180EC` 或所选基调）协调
- 白字区域需保证对比度，避免浅色氛围图影响可读性
- 若首屏为深色底（如稳重深蓝），人物与图标宜偏亮或带描边以保证识别

## 5. 色彩与风格约定

### 5.1 主色参考（与配色规范一致）

| 基调 | 主色 | 用途 |

- 图标种类过多（同屏建议 ≤ 3 个）

### 6.2 背景色与头图颜色融合（强制）

**约定**：头图背景色须根据加载的图标/人物色调动态适配，与画面风格融合。

**色彩调用原则（与《且慢营销配色规范》一致）**：

assets 内图片的主色须**映射至配色规范色板**，背景渐变须选用同一色系或邻近色系，确保**视觉柔和、过渡自然**。禁止使用与图片主色无关或对比过强的背景色。

| 图片主色/分类 | 配色规范引用 | 背景渐变建议 | 过渡要点 |
|---------------|--------------|--------------|----------|
| 暖色系（红金、橙黄、新春图标） | 新春红金 100–200、Gold 100 | `#FFF7E5 → #FEEBE7 → #FFF7E5` 或 `#FFF7E5 → #FCFDFD → #FFF7E5` | 渐变角度 165deg，中间色与图片主色同暖调 |
| 冷色系（蓝、青、紫） | 品牌蓝 A100–A200、科技紫 100–200 | `#F5F9FF → #DBEBFF → #F5F9FF` 或 `#F5F0FF → #E7DBFF → #F5F0FF` | 浅色起止、中间适度加深，避免硬切 |
| 人物（浅蓝/中性衣） | 品牌蓝 A100–A200 | `#F5F9FF → #DBEBFF → #F5F9FF` | 与人物浅蓝 polo/衬衫呼应 |
| 人物（紫衣） | 科技紫 100–200 | `#F5F0FF → #E7DBFF → #F5F0FF` | 与紫蓝西装/上衣同色系 |
| 人物（深蓝衣） | 稳重深蓝 B100–B200 | `#F0F6FF → #DBEBFF → #F0F6FF` | 浅蓝底不压人物 |
| 人物（暖色衣） | Gold 100 + Neutral 100 | `#FFF7E5 → #FCFDFD → #FFF7E5` | 米黄、浅棕与暖金 100 融合 |
| 风景（蓝绿、中性） | Neutral 100 + A100 | `#FCFDFD → #F5F9FF → #FCFDFD` | 低饱和，不抢主体 |

**过渡自然要点**：

- **起止色相近**：渐变起止尽量为同色或同档位（如 100↔100），中间色可适度加深（100→200→100），形成柔和呼吸感。
- **色相一致**：背景与图片主色保持同一色相或邻近色相（蓝↔青↔紫、红↔金↔橙），避免冷暖混搭。
- **饱和度克制**：头图背景统一使用 100–200 档浅色，不引入 300 及以上深色，保证文字可读与画面清爽。
- **渐变角度**：建议 `165deg` 或 `180deg`，自左上至右下或自上而下，与常见阅读视线一致。

**实现方式**：
1. 引入 `assets/assets-config.js`
2. 使用 `ASSETS.getRandomWithBg(theme)` 随机取图，并获取 `bgTheme`
3. 将 `bgTheme` 写入 header 的 `data-bg` 属性：`header.setAttribute("data-bg", res.bgTheme)`
4. 在 CSS 中为 `[data-bg="cny"]`、`[data-bg="cute"]`、`[data-bg="people"]`、`[data-bg="people-purple"]`、`[data-bg="people-deep"]`、`[data-bg="people-warm"]`、`[data-bg="cool"]`、`[data-bg="warm"]`、`[data-bg="landscape"]` 等分别定义对应渐变背景（与《且慢营销配色规范》一致；人物类按衣服主色选用 people/people-purple/people-deep/people-warm）

**bgTheme 与背景示意**：cny→红金、cute→金+中性、people→品牌蓝（人物浅蓝/中性衣服）、people-purple→科技紫（人物紫衣）、people-deep→稳重深蓝（人物深蓝衣）、people-warm→暖金（人物暖色衣）、cool→冷蓝、warm→暖金、landscape→中性+蓝。**人物类须按衣服主色选用对应 people-* 或 people**。
